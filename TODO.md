## Open

- [ ] **vibe: `/copy` + `vibe-copy` OSC 52 clipboard bridge** — inside-container exact-byte copy to the Mac clipboard via the OSC 52 escape sequence, so code Claude produces can be pasted elsewhere without the rendering-padding mangling that bites terminal-selection copy. Two surfaces: (a) `vibe-copy` shell helper on `$PATH` inside the image — reads stdin, base64-encodes, emits `\033]52;c;<BASE64>\a`, respects the terminal's payload size limit (handle oversize gracefully — chunk or refuse with a clear message); (b) `/copy` slash command synced via `install-claude-extras.sh` — Claude writes the last code block it produced to a temp file and pipes through `vibe-copy`. Documented target terminal: **Ghostty** (OSC 52 on by default). Also works: iTerm2 (toggle in Prefs → General → Selection), Kitty, Alacritty, WezTerm. **Won't** work: Apple Terminal.app and other stock terminals lacking OSC 52 — fallback path always writes to a known bind-mounted scratch file (`/workspace/.vibe/copy-latest.txt`) so Terminal.app users can still `cat` or `pbcopy` it from the host. Security notes: OSC 52 write-only, no read (read is a reasonable infoleak vector and most terminals gate it off); payload size limits vary (iTerm2 configurable, Ghostty generous, Kitty default 10KB — document). Open design questions: how does `/copy` identify "the last code block" unambiguously (parse Claude's own message history? take optional argument like `/copy <block-label>`?); test strategy for OSC 52 — capture stdout bytes from a fake terminal, assert the escape sequence is well-formed and base64 decodes back to input.
